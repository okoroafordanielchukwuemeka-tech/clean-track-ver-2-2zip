# PHASE 2 — Trial Expiration Automation Audit + Implementation

Current problem:

Users can remain on trial indefinitely.

Fix the subscription lifecycle.

Required flow:

New signup:

trial
↓
14 days active
↓
trial expiration
↓
past_due
↓
grace period
↓
suspended

Implement:

A scheduled background job.

Requirements:

Daily check:

Find laundries where:

subscriptionStatus = trial

AND

trialEndsAt < current date

Action:

Change:

subscriptionStatus:
trial → past_due

Create subscription_logs entry:

fromStatus:
trial

toStatus:
past_due

reason:
trial_expired

# PHASE 3 — Past Due Enforcement

Audit current requireOperational middleware.

Determine:

Does past_due users still create:

* orders
* workers
* branches
* payments

Implement correct behavior.

Recommended:

Past_due accounts should have:

Grace period:
7 days

During grace period:

Allow normal usage.

After 7 days:

Change:

past_due → suspended

Suspended accounts:

Can:

* login
* view data
* upgrade plan

Cannot:

* create new orders
* create workers
* create branches

Every transition must create subscription_logs entry.
