PHASE 7.2 — DOCUMENT GENERATION AUDIT & STANDARDIZATION

IMPORTANT

Do NOT build unrelated features.

Do NOT modify business logic unless required to correct document accuracy.

This phase focuses on making every printable/exportable document in CleanTrack production-ready and consistent.

---

STEP 1 — Audit Existing Documents

Audit every document currently generated by the system.

Include:

* Order Receipt
* Payment Receipt
* Customer Statement
* Printable pages
* PDF exports
* Browser print layouts

Determine:

1. Which documents already exist.
2. Which use different calculations.
3. Whether any duplicate calculation logic exists.
4. Whether all documents use the same financial engine.
5. Whether document totals always match database values.
6. Whether document layouts are responsive.
7. Whether branding is consistent.

---

STEP 2 — Financial Consistency

Verify every document uses exactly the same values as the verified Customer Statement engine.

Confirm:

* Base price
* Extra charges
* Discounts
* Payments
* Running balances
* Outstanding balance
* Cancelled orders
* Voided payments

There must be one single source of truth.

No duplicated financial calculations.

---

STEP 3 — Receipt Audit

Verify Order Receipt contains:

* Laundry logo (when available)
* Business name
* Branch
* Receipt number
* Order ID
* Customer
* Phone
* Services
* Quantities
* Unit prices
* Discounts
* Extra charges
* Total
* Outstanding balance
* QR / barcode placeholder
* Date
* Staff member

Verify Payment Receipt contains:

* Payment number
* Payment date
* Customer
* Order
* Amount paid
* Remaining balance
* Payment method
* Staff
* Business branding

Verify formatting is professional.

---

STEP 4 — Customer Statement PDF

Verify PDF includes:

Opening Balance

Chronological Ledger

Running Balance

Charges

Payments

Cancelled Orders

Credits

Closing Balance

Summary section including:

* Total Charges
* Total Payments
* Discounts
* Extra Charges
* Cancelled Orders
* Closing Balance

Verify PDF exactly matches the verified statement API.

---

STEP 5 — Pickup Receipt

Determine whether a Pickup Receipt exists.

If missing:

Design and implement one.

Include:

* Pickup Number
* Customer
* Order
* Items collected
* Remaining items
* Outstanding balance
* Staff
* Date
* Customer acknowledgement area

---

STEP 6 — Print Experience

Audit browser printing.

Verify:

* A4 layout
* Thermal printer compatibility (58mm / 80mm where possible)
* No clipped content
* No overlapping elements
* Consistent spacing
* Proper page breaks

---

STEP 7 — Branding

Standardize all documents.

Use:

* Same typography
* Same colours
* Same spacing
* Same header
* Same footer

Ensure every document looks like part of one professional product.

---

STEP 8 — Verification

Run live tests.

Generate:

* Order Receipt
* Payment Receipt
* Customer Statement
* Pickup Receipt (if implemented)

Verify:

* Totals match database
* PDFs render correctly
* Browser print works
* Mobile print works
* No calculation differences

---

Provide:

1. Existing Documents
2. Missing Documents
3. Financial Consistency Audit
4. Receipt Audit
5. PDF Audit
6. Print Audit
7. Branding Improvements
8. Failed Tests
9. Fixed Issues
10. Remaining Risks
11. Production Readiness Score

Do not begin WhatsApp templates, Academy, Marketing AI, or new customer-facing features until all document generation is production-ready.
