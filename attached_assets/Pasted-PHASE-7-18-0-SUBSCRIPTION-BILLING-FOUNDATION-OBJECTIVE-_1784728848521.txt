PHASE 7.18.0 — SUBSCRIPTION & BILLING FOUNDATION

OBJECTIVE

Build the complete subscription and billing system for CleanTrack.

This phase establishes how laundries subscribe, pay, renew, upgrade, downgrade, and how subscription capacity is enforced.

DO NOT begin Railway deployment.

DO NOT begin WhatsApp integration.

DO NOT begin AI Marketing.

Focus ONLY on subscription and billing.

────────────────────────────

CORE PHILOSOPHY

CleanTrack does NOT sell features.

Every paying customer receives the COMPLETE CleanTrack experience.

No operational feature should ever be disabled because of the selected plan.

Plans differ ONLY by business capacity.

Examples:

• Number of branches

• Number of workers

• Monthly order volume

• Storage

• WhatsApp usage

• Future AI credits

Never implement feature locking.

Implement capacity limits only.

The goal is:

A small laundry using the Starter plan should enjoy the exact same software experience as a large laundry using Professional or Enterprise.

────────────────────────────

STEP 6 — Capacity Enforcement

Create middleware that only enforces capacity.

Examples

Worker Limit

When exceeded:

Display:

"You have reached your worker limit for the Starter plan."

Offer:

Upgrade Plan

Branch Limit

Prevent creating another branch.

Order Limit

Prevent creating additional orders once the monthly allocation has been reached.

Always explain WHY.

Never hide functionality.
