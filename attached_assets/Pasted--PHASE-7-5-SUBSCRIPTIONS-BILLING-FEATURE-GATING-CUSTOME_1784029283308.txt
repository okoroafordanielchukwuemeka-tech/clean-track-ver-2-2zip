# PHASE 7.5 — SUBSCRIPTIONS, BILLING, FEATURE GATING & CUSTOMER LIFECYCLE

## IMPORTANT

Do NOT build anything outside this scope.

Do NOT modify existing laundry workflows unless required for billing integration.

Do NOT remove existing functionality.

First perform a COMPLETE audit before implementing anything.

---

# STEP 1 — AUDIT CURRENT SYSTEM

Audit the current application and determine:

1. Is there already a subscription system?
2. Is there already a billing table?
3. Are plans hardcoded?
4. Are features currently gated?
5. Can users currently exceed any limits?
6. Is WhatsApp already integrated?
7. Are there existing email templates?
8. Is onboarding email already implemented?
9. Is trial logic already present?
10. Are there existing cron jobs or scheduled tasks?

Provide findings before implementation.

---

# STEP 2 — DESIGN THE BILLING ARCHITECTURE

Design the subscription system so it can easily scale in the future.

Support:

* Free Trial
* Monthly Billing
* Future Annual Billing

The system must NOT require code changes when adding new plans later.

Plans must be database-driven.

Create tables if missing.

Suggested entities:

* Plans
* Subscriptions
* Subscription History
* Feature Limits
* Usage Counters
* Billing Events
* Payment Records

# STEP 4 — FEATURE GATING

Every premium feature must check subscription before execution.

Never hide features only in the frontend.

Enforce limits in BOTH:

* Backend
* Frontend

Example:

Starter attempting:

Create third worker

Return:

403

"Your current plan allows only 2 workers."

Professional attempting:

Create sixth branch

Return:

403

"Upgrade to Enterprise."
