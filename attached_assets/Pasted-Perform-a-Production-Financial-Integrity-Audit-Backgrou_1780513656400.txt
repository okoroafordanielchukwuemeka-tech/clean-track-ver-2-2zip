Perform a Production Financial Integrity Audit.

Background:

The Identifier Safety Audit found two remaining production risks:

1. receiptNumber generation currently uses MAX()+1 with retry logic.
2. idempotency middleware can allow duplicate processing when two requests with the same Idempotency-Key arrive simultaneously.

These risks directly affect payments and financial records.

The goal is to eliminate all realistic scenarios where duplicate payments, duplicate pickups, or duplicate financial records can be created.
