Perform an Alerting & Operational Incident Management Audit.

Mission:

Build the operational alert layer for CleanTrack.

The goal is:

When something important breaks, degrades, becomes risky, or requires attention, the system should automatically create an alert that owners and CleanTrack admins can see.

Do NOT implement notifications yet.

Do NOT implement email.

Do NOT implement WhatsApp.

Do NOT implement SMS.

Only build the alerting foundation.

==================================================
PHASE A — AUDIT
===============

Audit all existing monitoring systems:

* Sync Health
* Device Heartbeats
* Recovery Center
* Backup Center
* Schema Checkpoints
* Operations Dashboard
* Worker Activity
* Audit Logs
* Payment Integrity Checks
* Queue Monitoring

For each system report:

Current Visibility
Current Detection
Current Blind Spots
Current Severity
Alert Opportunities

PHASE C — ALERT RULES
=====================

Create alert rules for:

1. Device offline > 30 minutes
2. Device offline > 24 hours
3. Pending queue > 500
4. Pending queue > 1000
5. Failed sync count > threshold
6. Conflict count > threshold
7. Backup missing
8. Backup verification failed
9. Schema checkpoint missing before migration
10. Outdated application version
11. Payment integrity mismatch
12. Order balance inconsistency
13. Repeated worker failures
14. Excessive queue growth
15. Heartbeat missing

For each rule provide:

Trigger
Severity
Auto Resolution Logic
Required Metadata
