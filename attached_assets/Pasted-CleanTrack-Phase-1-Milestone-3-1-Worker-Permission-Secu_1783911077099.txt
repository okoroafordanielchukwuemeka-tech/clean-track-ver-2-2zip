CleanTrack — Phase 1 Milestone 3.1

Worker & Permission Security Fixes

We have completed the audit.

Do NOT redesign the worker system.

Do NOT add new features.

Implement ONLY the following two improvements.

---

## FIX 1 — Protect Owner Financial Data

Workers must NEVER have access to owner financial information.

Audit every analytics endpoint.

Ensure workers CANNOT access:

* Total expenses
* Salary expenses
* Utility expenses
* Rent
* Profit
* Overall financial summaries
* Any owner-only financial metrics

Workers should only receive operational analytics relevant to their own branch, such as:

* Orders
* Ready orders
* Processing orders
* Pickups
* Customer counts
* Operational workload

Verify:

* API returns 403 or removes owner-only fields.
* No financial information leaks through any endpoint.
* Owners still receive the full analytics dashboard.

---

## FIX 2 — New Worker Onboarding

Currently, a new worker with no permissions sees an empty Worker Station.

This creates confusion.

Implement a proper onboarding state.

When a worker logs in and has no assigned permissions:

Display a clear screen explaining:

"Your account is active but your manager has not assigned work permissions yet."

Include:

* Friendly illustration or icon
* Status message
* Instruction to contact the owner
* No broken or empty dashboard

Do NOT show blank screens.

---

## Verification

After implementation:

1. Create a new worker.

4. Verify the onboarding screen appears.
