PHASE 7.11.1 — CLOUDINARY IMAGE SYSTEM CERTIFICATION

IMPORTANT

Do NOT implement new features.

Do NOT begin Phase 8.

This phase is strictly a verification, bug-fixing, and certification pass for the new Cloudinary image system.

Assume nothing works until it has been tested.

Every test must be performed using the real frontend, real backend, real database, and the live Cloudinary account already configured.

---

## OBJECTIVE

Completely validate the Cloudinary migration and ensure there are no broken image workflows anywhere in CleanTrack.

Fix every issue discovered before completing this phase.

---

## SECTION 1 — Upload Workflow

Perform a complete frontend test.

Create a brand-new service.

Upload:

* JPG
* PNG
* WEBP

Verify:

* upload succeeds
* preview appears immediately
* image displays on the Services page
* image persists after page refresh
* Cloudinary URL is saved in the database
* thumbnail URL is generated correctly
* no browser console errors
* no failed network requests

Provide screenshots.

---

## SECTION 2 — Replace Image

Choose an existing service.

Replace its image.

Verify:

* old image disappears
* new image appears immediately
* thumbnail updates
* browser cache does not show stale images
* database points to the new Cloudinary asset
* previous Cloudinary asset is deleted

Confirm no orphaned image remains.

Provide before/after screenshots.

---

## SECTION 3 — Duplicate Service

Duplicate a service that already contains an image.

Verify the intended behaviour.

If the product intentionally shares the same Cloudinary asset:

* confirm this is documented.

If duplicates should have independent images:

* verify a new Cloudinary asset is created.

Ensure no broken image links.

Provide screenshots.

---

## SECTION 4 — Archive / Restore

Archive a service with an image.

Verify:

* image still renders correctly in archived view
* restore returns image correctly
* image URL never changes
* no missing thumbnails

Provide screenshots.

---

## SECTION 5 — Delete Service

Delete a service containing an image.

Verify:

* database record removed
* Cloudinary asset deleted
* thumbnail removed
* no orphan assets remain
* no broken links remain

Confirm deletion using the Cloudinary API.
