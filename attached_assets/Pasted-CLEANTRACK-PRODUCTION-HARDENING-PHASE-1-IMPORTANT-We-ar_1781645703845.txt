PHASE B — Password Recovery System

Currently owners can permanently lose access if they forget passwords.

Implement a secure password reset system.

Requirements:

Owner clicks:

"Forgot Password"

↓

Enter email

↓

Receive reset link

↓

Create new password

↓

Login normally


Security requirements:

- Token must expire
- Token must be single-use
- Store hashed reset tokens
- Never store raw reset tokens
- Rate limit reset requests
- Prevent email enumeration attacks


Create:

password_reset_tokens table


Test:

Successful reset

Expired token

Used token

Invalid token

Multiple requests
